# 协作默认规则

## 适用范围

本规则适用于 Agent 在该 Obsidian Vault 中的默认协作方式。

## 读取权限

- Vault 内所有文件均可自由读取、搜索、分析和总结。
- 读取行为不需要额外确认。

## 自由编辑区

- `Agent文档/` 是 Agent 的自由新增、修改、删除区域。
- Agent 可在该目录中创建协作记录、草稿、变更方案、补丁说明和中间产物。
- 除非用户另有说明，该目录中的内容可由 Agent 自主管理。

## 非自由编辑区变更流程

当需要对 `Agent文档/` 以外的 Vault 内容进行新增、修改或删除时，默认流程如下：

1. 先检查当前 git 状态。
2. 先将当前状态推送到远端仓库，确保远端保存变更前快照。
3. 将当前状态更新到 IMA，确保 IMA 中也保存变更前状态。
4. 将拟变更内容、patch、文件清单或产物草稿先放入 `Agent文档/`。
5. 等待用户明确批准。
6. 获批后再修改 `Agent文档/` 以外的 Vault 内容。
7. 修改完成后汇报变更内容、涉及文件和验证情况。

## IMA 同步要求

- 对 `Agent文档/` 以外的 Vault 内容进行新增、修改或删除前，除了推送 GitHub，还必须先完成 IMA 更新。
- IMA 更新应使用已安装的 `ima-skill` 执行。
- 执行 IMA 更新前需要可用的 IMA OpenAPI 凭证：`IMA_OPENAPI_CLIENTID` 和 `IMA_OPENAPI_APIKEY`，或 `IMA_CLIENT_ID` 和 `IMA_API_KEY`，或 `~/.config/ima/client_id` 与 `~/.config/ima/api_key`。
- 如果 IMA 凭证缺失、IMA 更新失败，或目标知识库不明确，应暂停对主 Vault 的修改，并先向用户说明阻塞原因。
- 未确认 IMA 已完成更新时，不进入主 Vault 实际修改步骤。

## 删除与覆盖

- 不删除 `Agent文档/` 以外的内容，除非用户明确要求或批准。
- 不覆盖用户已有内容，除非用户明确要求或批准。
- 如果发现工作区存在非 Agent 产生的改动，应保留并协同处理，不擅自回退。

## 远端仓库

- 默认远端仓库为 `https://github.com/dsaj4/Vault`。
- 当前本地分支默认为 `main`。
- 推送或同步时优先保持本地与远端状态清晰一致。
